Complete Project Explanation for Your Presentation
1. PROJECT OVERVIEW
Resume Analyzer is an AI-powered web application that:

Accepts PDF/DOCX resume uploads
Uses AI (Groq API with Llama model) to analyze resumes
Provides ATS (Applicant Tracking System) scores
Detects skills (technical, soft, tools)
Compares resumes against job descriptions
Gives improvement suggestions

3. WHY NEXT.JS vs REACT.JS
React.js (Plain)
Just a library for building UI
You need to add routing yourself (react-router)
You need to set up your own server for API
All pages load on client side (slow initial load)
SEO is bad (search engines can't see content easily)
Next.js (Framework built on React)
Feature	Benefit
File-based Routing	Create app/about/page.tsx → automatically get /about route
API Routes	Create app/api/analyze/route.ts → automatically get /api/analyze endpoint
Server-Side Rendering (SSR)	Pages render on server, send complete HTML to browser → Fast, SEO-friendly
Built-in Image Optimization	Automatic image resizing and lazy loading
Zero Configuration	Works out of the box, no webpack config needed
Full-Stack in One Project	Frontend AND backend in same codebase
Simple Answer for Presentation:

"I used Next.js because it gives me routing, API endpoints, and faster loading out of the box. With plain React, I would need to install many extra libraries and configure them manually."

4. SSR vs CSR (VERY IMPORTANT QUESTION!)
CSR (Client-Side Rendering) - Traditional React
Problem: User sees blank screen while JavaScript loads. Search engines see empty page.

SSR (Server-Side Rendering) - Next.js
Benefits:

Faster First Paint: User sees content immediately
Better SEO: Search engines can read the HTML
Better for slow devices: Less work for user's browser
Visual Comparison:
In Your Project:
"use client" = This component runs on CLIENT (CSR)
No directive = Component runs on SERVER (SSR)
API routes (route.ts) = Always run on server

6. POTENTIAL Q&A FOR YOUR PRESENTATION
Q: "How does the file upload work?"
A: "I use react-dropzone for drag-and-drop. When user drops a file, it creates a File object. I send this to my API using FormData. The API parses the file using pdfjs-dist for PDFs or mammoth for DOCX."

Q: "How does the AI analysis work?"
A: "I use the Groq API with the Llama 3.1 8B model. I send the resume text with a structured prompt. The AI returns a JSON with scores, skills, and suggestions. This is called a Large Language Model (LLM)."

Q: "What is 'use client' at the top of files?"
A: "In Next.js 13+, all components are Server Components by default. If I need useState, useEffect, or any browser-only features (like localStorage), I add 'use client' to make it a Client Component."

Q: "What is Context API (ThemeContext)?"
A: "Context API is React's built-in way to share data between components without passing props through every level. I use it for theme (dark/light mode) so any component can access and change the theme."

Q: "Why Framer Motion for animations?"
A: "Framer Motion is a React animation library. It's easier than CSS animations for complex interactions. I use it for page transitions, hover effects, and loading animations. Example: <motion.div animate={{ opacity: 1 }}>"

Q: "What is an API route in Next.js?"
A: "Files in app/api/ folder become backend endpoints. app/api/analyze/route.ts creates /api/analyze. I can write server-side code there (access databases, call external APIs, process files)."

Q: "What is glassmorphism?"
A: "It's a modern design trend where UI elements look like frosted glass. I achieve this with backdrop-filter: blur() and semi-transparent backgrounds. You can see it in my glass-card class."

Q: "How do you handle dark/light mode?"
A: "I use React Context to store the theme state. When user toggles, I update localStorage and add/remove dark/light class on <html>. CSS then applies different colors based on this class."

Q: "What is the App Router vs Pages Router?"
A: "Next.js 13+ introduced App Router (app/ folder). It has better layouts, server components, and streaming. Pages Router (pages/ folder) is the older way. I use App Router - it's the modern approach."
